# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it through GitHub's Security Advisories feature:

1. Navigate to the **Security** tab of this repository
2. Click on **Report a vulnerability**
3. Fill out the vulnerability report form with as much detail as possible

**Please do NOT:**
- Open a public GitHub issue for security vulnerabilities
- Disclose the vulnerability publicly before it has been addressed

## What to Include in Your Report

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if available)

## Response Timeline

- **Initial Response:** Within 48 hours of submission
- **Status Update:** Within 7 days with an assessment and remediation timeline
- **Resolution:** We aim to resolve confirmed vulnerabilities within 30 days

## Supported Versions

We provide security updates for the latest release only. Please ensure you are using the most recent version.

## Scope

This security policy applies to the code in this repository. Third-party dependencies are managed separately but will be updated promptly when security patches are available.
